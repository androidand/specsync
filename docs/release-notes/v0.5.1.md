# specsync v0.5.1

v0.5.1 completes and hardens the lifecycle work prematurely published as v0.5.0.

- Changes move from `stage:active` to `stage:complete` when every task is checked.
- `-close-completed` closes completed tracker items and reopens them when new unchecked work appears.
- Completion derived from reconciled GitHub checkboxes is projected in the same sync.
- The npm installer verifies release archives against `checksums.txt`, stages concurrent installs separately, and preserves an existing binary when an upgrade fails.
- Bundled skills for Claude Code, Codex, OpenCode, Copilot, and generic agents stay synchronized from one canonical source.

The release also corrects documentation around dry-run support and removes claims for features that are not shipped.
